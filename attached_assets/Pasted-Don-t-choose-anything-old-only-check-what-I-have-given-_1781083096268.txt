Product Name

Overthinkers

One-Line Product Vision

A social platform where people anonymously or publicly share thoughts, emotions, overthinking moments, relatable experiences, polls, and discussions in a safe and engaging community.

Most social media platforms reward attention-seeking content.

People often have:

random thoughts

deep emotions

overthinking moments

life questions

relationship confusion

career stress

mental clutter


but no dedicated place to express them.

Overthinkers solves this.

DOCUMENT 7 — AI DEVELOPMENT RULES (CRITICAL FOR REPLIT AGENT)

Ye document har prompt ke saath dena chahiye.

Mandatory Rules

1. Analyze entire codebase before coding.


2. Create implementation plan before changes.


3. Identify already existing features.


4. Never duplicate existing functionality.


5. Never create duplicate database tables.


6. Never modify auth system without dependency analysis.


7. Never delete code unless verified unused.


8. Search entire project before creating new screens.


9. Reuse existing components whenever possible.


10. Preserve backward compatibility.


11. Fix root cause, not symptoms.


12. Generate change report before implementation.


13. Generate completion report after implementation.


14. If feature already exists:



Do not rebuild.

Improve existing implementation.


15. If schema already exists:



Extend safely.

Never recreate.


16. Before every change verify:



UI impact

API impact

Database impact

Auth impact

Realtime impact


17. Complete one phase fully before starting another.


18. Never make assumptions when code already contains answers.


19. Validate all TypeScript errors before completion.


20. Project stability is higher priority than adding new features.
